# 快速开始

零起步，几分钟之内就能拥有一台 hams 管起来的工作站。

## 装 hams

挑一个顺手的方式：

<Tabs items={['curl（推荐）', 'Homebrew', 'GitHub Releases']}>
  <Tabs.Tab>
```bash
bash -c "$(curl -fsSL https://github.com/zthxxx/hams/raw/master/scripts/install.sh)"
```

脚本会识别你的平台（darwin/arm64、linux/amd64、linux/arm64），下最新版的二进制放到 `/usr/local/bin/hams`，
同时记一条 channel 标记，之后 `hams self-upgrade` 就知道该从哪升。
  </Tabs.Tab>
  <Tabs.Tab>
```bash
brew install zthxxx/tap/hams
```

走的是官方 Homebrew tap。以后 `hams self-upgrade` 会自动调 `brew upgrade hams`，跟你日常的 brew 更新习惯一致。
  </Tabs.Tab>
  <Tabs.Tab>
从 [GitHub Releases](https://github.com/zthxxx/hams/releases/latest) 下自己平台的二进制：

| 平台 | 架构 | 文件 |
|------|------|------|
| macOS | Apple Silicon（M 系列） | `hams_darwin_arm64` |
| Linux | x86-64 | `hams_linux_amd64` |
| Linux | ARM64 | `hams_linux_arm64` |

```bash
# 以 macOS Apple Silicon 为例
curl -Lo hams https://github.com/zthxxx/hams/releases/latest/download/hams_darwin_arm64
chmod +x hams
sudo mv hams /usr/local/bin/hams
```
  </Tabs.Tab>
</Tabs>

装完确认一下版本：

```bash
hams --version
# hams v0.1.0 (commit: abc1234, built: 2026-04-12)
```

## 装点东西试试

用法很朴素 —— 把原来的命令前面加一个 `hams` 就行：

```bash
# 走 Homebrew 装 htop（比 top 好用的那个），顺手记到 Hamsfile
hams brew install htop

# 走 pnpm 全局装 serve，顺手记
hams pnpm add serve

# 配 git 身份，顺手记 —— `hams git` 接受原生 git 语法
hams git config user.name "Your Name"
hams git config user.email "you@example.com"

# VS Code 扩展也能记
hams code install ms-python.python
```

每条命令背后，hams 默默做三件事：

1. 真调底下的工具（`brew install htop`、`pnpm add -g serve`、`git config --global user.name ...`、`code --install-extension ...`）
2. 把这次安装写进对应的 Hamsfile（`default/Homebrew.hams.yaml`、`default/pnpm.hams.yaml`、`default/git-config.hams.yaml`、`default/vscodeext.hams.yaml`）
3. **首次运行时自动初始化 store** —— 在 `~/.local/share/hams/store/` 帮你 `git init`、写好 `.gitignore` 与 `hams.config.yaml`。无需先跑 `hams store init`。

CI 脚本如希望恢复旧的 "no store directory configured" 报错（不要自动初始化），设置 `HAMS_NO_AUTO_INIT=1` 即可。

## 在新机器上把一切恢复

这是 hams 真正发力的地方。面对一台全新的机器：

<Steps>
### 先把 hams 自己装上

```bash
bash -c "$(curl -fsSL https://github.com/zthxxx/hams/raw/master/scripts/install.sh)"
```

hams 二进制里内置了 go-git，所以哪怕这机器连 git 命令都还没有也没关系。

### 从 store 仓库 apply（一条命令）

```bash
# 一条命令开箱：拉 store、选 macOS profile、直接 apply。
# 缺失的 ~/.config/hams/hams.config.yaml 会自动创建。
hams apply --from-repo=<your-username>/hams-store --tag=macOS

# 也可以同时让 hams 在前置工具（Homebrew / pnpm / ansible）缺失时自动装：
hams apply --bootstrap --from-repo=<your-username>/hams-store --tag=macOS
```

把 `<your-username>/hams-store` 换成你自己的 GitHub 用户和 store 仓库名。
交互终端下 `--bootstrap` 不是必传 —— hams 跑到前置工具缺失的 provider 时会主动
弹 `[y/N/s]` 让你确认。CI / 脚本里记得显式传 `--bootstrap`（或 `--no-bootstrap`
让它前置缺失时直接报错退出）。

### 不需要交互式问问题

如果 `~/.config/hams/hams.config.yaml` 还不存在，hams 第一次运行时会自动写一份：
`tag: default` + 由主机名衍生的 `machine_id`。`--tag=<your-profile>` 可以临时覆盖默认 tag；
要持久化用 `hams config set tag <name>` 即可。

旧版本里那个交互式 `Profile tag / Machine ID` 提示现在只在 `hams store init` 中保留 ——
仅在你想把 store 放在 `${HAMS_DATA_HOME}/store/` 之外的目录、并手工准备时才用得上。

### 输 sudo 密码（如果需要）

如果你的 profile 用到了需要提权的 provider（比如 Linux 上的 `apt`），hams 会在开始前一次性问密码：

```
Some providers require sudo. Enter your password to continue:
Password:
```

密码输一次就好。hams 在后台每 4 分钟刷一下 sudo 票据，保证 apply 跑到一半不会再跳出来打扰你。

## 把改动推回去

装完新东西记得把 Hamsfile 推回 store，其他机器才能用上：

```bash
# 站在 store 仓库目录下
hams store push
# 自动暂存所有 Hamsfile 改动 → 提交 → 推到远端
```

不想用 hams 管 git 也行，Hamsfile 是纯 YAML，`git add` + `git commit` + `git push` 随便你。
