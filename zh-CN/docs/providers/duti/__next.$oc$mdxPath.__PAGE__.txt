# duti Provider

包装 [duti](https://github.com/moretension/duti) 来管 macOS 上"哪个文件类型用哪个应用打开"的关联关系。
比如你想让所有 `.md` 默认用 VS Code 打开、所有 `.ts` 用 Cursor、所有图片用 IINA 预览 —— 就是这个 provider 干的事。

**支持平台：** 仅 macOS

## 怎么用

```bash
# 把 TypeScript 文件默认用 VS Code 打开
hams duti set com.microsoft.VSCode public.typescript editor

# 列出被管的关联
hams duti list

# 取消一条关联（会恢复成系统默认）
hams duti unset com.microsoft.VSCode public.typescript editor
```

三个位置参数的意思：**应用 bundle ID**、**UTI（文件类型）**、**角色**（`viewer` / `editor` / `shell` / `all`）。

## Hamsfile 长什么样

```yaml
# macOS/duti.hams.yaml
schema_version: 1
provider: duti

groups:
  - tag: code
    items:
      - urn: "urn:hams:duti:vscode-ts"
        step: Open TypeScript files with VS Code
        bundle_id: com.microsoft.VSCode
        uti: public.typescript
        role: editor

      - urn: "urn:hams:duti:vscode-md"
        step: Open Markdown with VS Code
        bundle_id: com.microsoft.VSCode
        uti: net.daringfireball.markdown
        role: editor

  - tag: media
    items:
      - urn: "urn:hams:duti:iina-video"
        step: Open video with IINA
        bundle_id: com.colliderli.iina
        uti: public.movie
        role: viewer
```

## 常用的 UTI

| 文件类型 | UTI |
|----------|-----|
| Markdown | `net.daringfireball.markdown` |
| TypeScript | `public.typescript` |
| JavaScript | `com.netscape.javascript-source` |
| JSON | `public.json` |
| 图片（通用） | `public.image` |
| 视频（通用） | `public.movie` |
| PDF | `com.adobe.pdf` |
| 纯文本 | `public.plain-text` |

想知道某个扩展名对应什么 UTI？跑 `duti -x md` 就会告诉你。

## 怎么找应用的 bundle ID

```bash
# 比如 VS Code
osascript -e 'id of app "Visual Studio Code"'
# com.microsoft.VSCode

# 或者 mdls 也行
mdls -name kMDItemCFBundleIdentifier /Applications/Visual\ Studio\ Code.app
```

## 状态探测

探测方式：对每条记录跑 `duti -x <ext>` 或 `duti -d <uti>`，拿回当前系统的默认应用，
跟 Hamsfile 里的 `bundle_id` 比对。

## 前提条件

新 Mac 没装 duti？可以让 hams 通过 Homebrew 帮你装。给 `apply` 加上 `--bootstrap`，
hams 会在你确认后跑 `brew install duti`：

```bash
hams apply --bootstrap --from-repo=you/hams-store
```

交互终端不带这个 flag 时，hams 会把命令打出来让你 `[y/N/s]` 确认。详见
[hams apply → 关于 bootstrap 提示](/docs/cli/apply#关于-bootstrap-提示)。
brew 本身也是 opt-in consent，新 Mac 的 `brew → duti` 链一条 `hams apply --bootstrap`
就能走完。

或者按常规用 Homebrew 声明 duti：

```yaml
# Homebrew.hams.yaml
- app: duti
  intro: Set default applications for file types.
```

<Callout type="tip">
  `duti` 改的是 macOS 的 Launch Services 数据库，有时候新关联要重启 Finder 或者整台机器才生效。
  apply 跑完没看到效果，先 logout 再 login 一次。
