# git clone Provider

管你希望"在本地某个固定路径有这个仓库"的 git 项目。典型用法：dotfiles 仓库、个人笔记、
常用的自建工具 —— 那些你总是希望在新机器上同一个位置能找到的代码。

**支持平台：** macOS、Linux

## 怎么用

```bash
# 添加一条：把远端仓库克隆到本地指定路径
hams git-clone add https://github.com/zthxxx/dotfiles ~/Projects/dotfiles

# 列出被管的仓库
hams git-clone list

# 移除（只是停止管理，不会删本地文件）
hams git-clone remove ~/Projects/dotfiles
```

## Hamsfile 长什么样

```yaml
# macOS/git-clone.hams.yaml
schema_version: 1
provider: git-clone

groups:
  - tag: personal
    items:
      - urn: "urn:hams:git-clone:dotfiles"
        step: Clone dotfiles repository
        remote: https://github.com/zthxxx/dotfiles.git
        local_path: ~/Projects/dotfiles
        default_branch: main

      - urn: "urn:hams:git-clone:notes"
        step: Clone personal notes
        remote: git@github.com:zthxxx/notes.git
        local_path: ~/Documents/notes
```

字段：

- `remote` — 仓库 URL，HTTPS 或 SSH 都行
- `local_path` — 本地路径，支持 `~/` 展开
- `default_branch` — 可选，克隆后 checkout 到哪个分支。不填就用远端默认

## 状态探测

探测方式：

1. 看 `local_path` 目录存不存在
2. 如果存在，跑 `git -C <local_path> remote get-url origin`，对比 URL 是否匹配 Hamsfile 里的 `remote`
3. 如果都对得上，状态 `ok`

**hams 不会**帮你 pull 最新代码 —— 每次 apply 都替你 pull 会拖慢速度也可能冲突。这件事交给你自己掌握节奏。

## 使用 go-git 而不是 `git` 命令

hams 二进制内置了 [go-git](https://github.com/go-git/go-git)，意味着**新机器就算没装 git 命令**，
hams apply 也能把仓库克隆下来。这对 bootstrap 场景很关键 —— 你不用先 `brew install git` 再开始用 hams。

## SSH 认证

如果 `remote` 是 SSH 形式（`git@github.com:...`），hams 会按顺序尝试：

1. `~/.ssh/id_ed25519`、`~/.ssh/id_rsa` 等常见路径
2. 跑起来的 ssh-agent 里的 key
3. 最后回退到提示你交互输入

新机器上常见的卡点是 SSH key 还没配好。解决办法：要么用 HTTPS URL（配合 `gh auth login` 或 PAT），
要么在 apply 之前先把 SSH key 搞定。

<Callout type="tip">
  git-clone 适合"我需要这个仓库在这里"的场景。如果你只是想跑一个安装脚本，用 `bash` provider 的
  `curl | bash` 更直接。git-clone 的意义在于**持续管理**这个仓库的存在和位置。
</Callout>
