# VS Code 扩展 Provider

包装 `code` 命令行工具来管 VS Code 的扩展。每次换新机器重新一个个点装扩展是最烦的事情之一，
这个 provider 就是为了让那个过程消失。

同样支持 Cursor、VSCodium 这类 VS Code fork —— 只要它们也提供了兼容的 CLI 就行，
通过 `hams.config.yaml` 里改 `code_ext.cli_command` 可以切。

**支持平台：** macOS、Linux

## 怎么用

```bash
# 装一个扩展（用 publisher.name 形式）
hams code-ext install esbenp.prettier-vscode

# 卸掉
hams code-ext uninstall esbenp.prettier-vscode

# 列出被管的扩展
hams code-ext list
```

扩展 ID 去哪找？打开 VS Code 扩展面板，右键任意扩展 → Copy Extension ID，就是 `publisher.name` 这个格式。

## Hamsfile 长什么样

```yaml
# macOS/vscodeext.hams.yaml
schema_version: 1
provider: code-ext

groups:
  - tag: language-support
    items:
      - app: golang.go
        intro: Rich Go language support.
      - app: rust-lang.rust-analyzer
        intro: Rust language server.
      - app: dbaeumer.vscode-eslint
        intro: ESLint integration.

  - tag: theme
    items:
      - app: esbenp.prettier-vscode
        intro: Prettier formatter for VS Code.
      - app: PKief.material-icon-theme
```

## 状态探测

探测方式：hams 跑 `code --list-extensions`，把输出按行切分就是当前装了哪些。
简单粗暴但可靠。

## 前提条件

机器上得能调到 `code` 命令。macOS 上装好 VS Code 还得手动在命令面板里运行一次
"Shell Command: Install 'code' command in PATH"。建议用 `bash` provider 自动化一下：

```yaml
- urn: "urn:hams:bash:link-vscode-cli"
  step: Link VS Code CLI
  run: ln -sf "/Applications/Visual Studio Code.app/Contents/Resources/app/bin/code" /usr/local/bin/code
  check: command -v code
```

## 用 Cursor / VSCodium 怎么办

在 `hams.config.yaml` 里加一段：

```yaml
providers:
  code-ext:
    cli_command: cursor   # 或者 codium
```

hams 就会用 `cursor --install-extension` 代替 `code --install-extension`。

<Callout type="info">
  VS Code 的用户设置 `settings.json`、键位绑定 `keybindings.json` 不归这个 provider 管 ——
  它们是普通 dotfile，请用 `file` provider 或者 git-clone 一个 dotfiles 仓库处理。
</Callout>
