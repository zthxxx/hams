# VS Code Provider (`hams code`)

Wraps VS Code's `code` CLI to manage extensions. Clicking through the extension store one at a
time on every new machine is one of life's lesser joys — this provider exists to make that moment
disappear.

The CLI verb is `hams code` (renamed from the legacy `hams code-ext` in 2026-04-17 — the
underlying provider keeps `code-ext` as its internal manifest name for back-compat with existing
state files). The on-disk hamsfile is `<profile>/vscodeext.hams.yaml` for the same back-compat
reason.

For Cursor or VSCodium, hams will ship a separate `cursor` provider rather than overloading
`hams code` with a `cli_command:` switch — the editor-specific CLI behaves differently enough
(extension marketplace, settings sync, telemetry endpoints) that conflating them was a footgun.

**Platforms:** macOS, Linux

## Usage

```bash
# Install (publisher.name form)
hams code install esbenp.prettier-vscode

# Remove
hams code uninstall esbenp.prettier-vscode

# See what's managed
hams code list
```

To find an extension's ID: open the VS Code extensions panel, right-click any extension → Copy
Extension ID. That `publisher.name` string is what you want.

## Hamsfile example

```yaml
# default/vscodeext.hams.yaml
schema_version: 1
provider: code-ext

groups:
  - tag: language-support
    items:
      - app: golang.go
        intro: Rich Go language support.
      - app: rust-lang.rust-analyzer
        intro: Rust language server.
      - app: dbaeumer.vscode-eslint
        intro: ESLint integration.

  - tag: theme
    items:
      - app: esbenp.prettier-vscode
        intro: Prettier formatter for VS Code.
      - app: PKief.material-icon-theme
```

## How state is probed

`code --list-extensions` prints one extension ID per line. hams reads that list. Blunt but works.

## Bootstrap

The machine needs to have `code` on the PATH. On macOS you have to manually run "Shell Command:
Install 'code' command in PATH" from the VS Code command palette after installing the app. Automate
it with a `bash` provider step:

```yaml
- urn: "urn:hams:bash:link-vscode-cli"
  step: Link VS Code CLI
  run: ln -sf "/Applications/Visual Studio Code.app/Contents/Resources/app/bin/code" /usr/local/bin/code
  check: command -v code
```

<Callout type="info">
  User settings (`settings.json`) and key bindings (`keybindings.json`) are out of scope for this
  provider — they're just dotfiles. Use the `file` provider, or clone a dotfiles repo with
  `hams git clone`.
</Callout>
