# 快速开始

从零开始，几分钟内拥有一台完全受管的工作站。

## 安装 hams

选择最适合你的安装方式：

<Tabs items={['curl（推荐）', 'Homebrew', 'GitHub Releases']}>
  <Tabs.Tab>
```bash
bash -c "$(curl -fsSL https://github.com/zthxxx/hams/raw/master/scripts/install.sh)"
```

此命令会为你的平台（darwin/arm64、linux/amd64、linux/arm64）下载最新版本的二进制文件，
放置在 `/usr/local/bin/hams`，并写入一个 channel 标记供 `hams self-upgrade` 使用。
  </Tabs.Tab>
  <Tabs.Tab>
```bash
brew install zthxxx/tap/hams
```

通过官方 Homebrew tap 安装。后续 `hams self-upgrade` 将使用 `brew upgrade hams` 进行更新。
  </Tabs.Tab>
  <Tabs.Tab>
从 [GitHub Releases](https://github.com/zthxxx/hams/releases/latest) 下载适合你平台的二进制文件：

| 平台 | 架构 | 文件 |
|------|------|------|
| macOS | Apple Silicon (M1/M2/M3/M4) | `hams_darwin_arm64` |
| Linux | x86-64 | `hams_linux_amd64` |
| Linux | ARM64 | `hams_linux_arm64` |

```bash
# 以 macOS Apple Silicon 为例
curl -Lo hams https://github.com/zthxxx/hams/releases/latest/download/hams_darwin_arm64
chmod +x hams
sudo mv hams /usr/local/bin/hams
```
  </Tabs.Tab>
</Tabs>

验证安装：

```bash
hams --version
# hams v0.1.0 (commit: abc1234, built: 2026-04-12)
```

## 你的第一次安装

安装好 hams 后，像使用底层工具一样使用它 -- 只需在前面加上 `hams`：

```bash
# 通过 Homebrew 安装 bat（更好的 cat）并自动记录
hams brew install bat

# 通过 pnpm 全局安装 serve 并自动记录
hams pnpm add serve

# 设置你的 git 用户配置并记录
hams git-config user.name "Your Name"
hams git-config user.email "you@example.com"
```

每条命令执行后，hams 会：
1. 运行底层工具（`brew install bat`、`pnpm add -g serve` 等）
2. 将安装记录到你的 Hamsfile 中（`macOS/Homebrew.hams.yaml`、`macOS/pnpm.hams.yaml` 等）

## 在新机器上恢复一切

这就是 hams 大显身手的时候。在一台全新的机器上：

<Steps>
### 安装 hams 本身

```bash
bash -c "$(curl -fsSL https://github.com/zthxxx/hams/raw/master/scripts/install.sh)"
```

hams 内置了 `go-git`，所以你不需要预先安装 git。

### 从你的 store 仓库运行 apply

```bash
hams apply --from-repo=<your-username>/hams-store
```

将 `<your-username>/hams-store` 替换为你的 GitHub 用户名和 store 仓库名。

### 回答 profile 提示

第一次在新机器上 apply 时，hams 会问两个问题：

```
No profile found for this machine.

Profile tag (e.g. macOS, openwrt, work-linux): macOS
Machine ID (unique name for this machine, e.g. MacbookProM5X): MacbookProM3

Creating profile directory: macOS/
Writing ~/.config/hams/hams.config.yaml...
```

**Profile tag** 选择使用你 store 仓库中的哪个目录（例如 `macOS/` 或 `openwrt/`）。
**Machine ID** 用于命名你的状态目录（`.state/MacbookProM3/`）。

### 输入 sudo 密码（如果需要）

如果你的 profile 包含需要提升权限的 provider（例如 Linux 上的 `apt`），hams
会在启动时提示你输入一次 sudo 密码：

```
Some providers require sudo. Enter your password to continue:
Password:
```

hams 会在后台每 4 分钟刷新 sudo 票据，这样在 apply 过程中不会再次提示你。


## 推送你的更改

安装软件包后，将你的 Hamsfiles 推送到 store 仓库，这样在其他机器上也能使用：

```bash
# 在你的 store 仓库目录下
hams store push
# 暂存所有 Hamsfile 更改，提交并推送到远程
```

或者手动管理 git -- Hamsfiles 是纯 YAML，完全可以手动编辑。
