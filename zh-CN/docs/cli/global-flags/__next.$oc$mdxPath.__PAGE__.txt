# 全局 Flag

全局 flag 是作用于所有命令的开关，位置固定放在 `hams` 和命令名**之间**：

```bash
hams [全局 flag] <命令> [参数]
```

比如 `hams --debug apply` 对，`hams apply --debug` 不行 —— 后者会被认为是传给 `apply` 的 flag。

## Flag 一览

| Flag | 类型 | 默认 | 说明 |
|------|------|------|------|
| `--debug` | bool | `false` | 打开 debug 级别日志，问题排查神器 |
| `--dry-run` | bool | `false` | 只打印计划，不真动手 |
| `--json` | bool | `false` | 输出 JSON，给脚本和 AI agent 用 |
| `--no-color` | bool | `false` | 关掉 ANSI 颜色码（输出到文件或不支持颜色的终端时用） |
| `--config=<path>` | string | `~/.config/hams/hams.config.yaml` | 指定另一份配置文件 |
| `--store=<path>` | string | 来自配置 | 覆盖 store 目录路径 |
| `--tag=<name>` | string | 来自配置（缺省为 `default`） | 当次执行使用的 profile tag。`--profile` 是兼容别名。优先级：`--tag` > 配置 `tag:` > `"default"` |
| `--help` | bool | `false` | 打印帮助 |
| `--version` | bool | `false` | 打印版本信息 |

## 常见用法

```bash
# debug 模式跑 apply，看 provider 里到底在做什么
hams --debug apply

# 真执行前先预览一遍
hams --dry-run apply

# 临时用另一个配置文件（比如测试新配置）
hams --config=/tmp/test-config.yaml apply

# 想把 list 结果给 jq 处理？来 JSON
hams --json list | jq '.[] | select(.state == "missing")'

# 临时切换 profile，不改全局配置
hams --tag=work-linux apply           # 推荐写法
hams --profile=work-linux apply       # 兼容旧写法
```

## 首次运行自动初始化

第一次执行 `hams apply` 或任意 provider 命令（`hams brew install jq`、`hams git config user.name X` 等）时，
hams 会自动：

1. 在 `~/.config/hams/hams.config.yaml` 写入默认配置（`tag: default` + 主机名衍生的 `machine_id`）
2. 在 `${HAMS_DATA_HOME}/store/`（默认 `~/.local/share/hams/store/`）创建 store —— `git init` 初始化、
   写入内置的 `.gitignore`（含 `.state/`、`*.local.*`）和占位 `hams.config.yaml`
3. 把 store 路径写回全局配置，供之后运行直接使用

如需关闭自动初始化（例如 CI 脚本希望保留旧的 "no store directory configured" 报错），设置环境变量 `HAMS_NO_AUTO_INIT=1` 即可。

## 为什么要区分"全局 flag"和"命令 flag"

说白了就是为了避开歧义。拿 `hams brew install htop --debug` 举例：`--debug` 到底是给 hams 还是给 `brew`？
hams 的办法是：全局 flag 必须在命令名之前，命令/provider 名之后的东西一律当作命令自己的参数。
这样规则清晰，hams 拦 hams 的，`brew` 拿 `brew` 的，谁也不误伤谁。

## `--dry-run` 的细节

开了 `--dry-run` 之后，hams 会：

- 照常跑 `refresh` 读取机器现状（这步没有副作用）
- 照常算 diff、打印计划
- **不**跑任何 provider 的 install/remove/write 操作
- **不**写 `.state/` 文件（下次真跑的时候状态不会对不上）
- 退出码照常 —— 有计划就 `0`，有冲突就非 `0`

所以它是真正的"干跑一次"，可以安全地用来审查自己的 Hamsfile 改动。
