# Provider Catalog

hams ships with 15 builtin providers covering the full spectrum of workstation configuration.
Each provider wraps an existing CLI tool and gives it auto-record, state tracking, and diff-based apply.

## Builtin Providers

| Provider | Wraps | Platforms | Description |
|----------|-------|-----------|-------------|
| [Homebrew](./homebrew) | `brew` | macOS, Linux | Formulae and casks via Homebrew |
| [apt](./apt) | `apt-get` | Linux (Debian/Ubuntu) | System packages via APT |
| [pnpm](./pnpm) | `pnpm` | macOS, Linux | Global Node.js packages via pnpm |
| [npm](./npm) | `npm` | macOS, Linux | Global Node.js packages via npm |
| [uv](./uv) | `uv` | macOS, Linux | Python tools via uv |
| [go](./go) | `go install` | macOS, Linux | Go binaries via `go install` |
| [cargo](./cargo) | `cargo install` | macOS, Linux | Rust binaries via Cargo |
| [vscode-ext](./vscode-ext) | `code` | macOS, Linux | VS Code extensions |
| [mas](./mas) | `mas` | macOS only | Mac App Store apps |
| [git-config](./git-config) | `git config` | macOS, Linux | Git configuration key-value pairs |
| [git-clone](./git-clone) | `git clone` | macOS, Linux | Repository clones at fixed local paths |
| [defaults](./defaults) | `defaults` | macOS only | macOS `defaults write` settings |
| [duti](./duti) | `duti` | macOS only | Default app associations for file types |
| [bash](./bash) | `bash` | macOS, Linux | Arbitrary shell scripts with idempotency checks |
| [Ansible](./ansible) | `ansible` | macOS, Linux | Ansible playbooks and ad-hoc tasks |

## Resource Classes

Every provider's resources fall into one of four classes, each with a different probe strategy:

### Package (native list)

Providers: Homebrew, apt, pnpm, npm, uv, go, cargo, mas, vscode-ext

Probe method: The provider runs its native list command (e.g., `brew list --formula`, `npm list -g`)
and checks whether the package name appears in the output. Version is recorded from the list output.

```yaml
- app: ripgrep
  intro: Fast recursive grep alternative.
```

### KV Config (read-back)

Providers: defaults, git-config, duti

Probe method: The provider reads back the current value using a read command (e.g., `defaults read`, `git config --get`)
and compares it to the desired value. If they match, the resource is `ok`.

```yaml
- urn: "urn:hams:defaults:finder-show-extensions"
  step: Show all file extensions in Finder
  domain: NSGlobalDomain
  key: AppleShowAllExtensions
  value: true
```

### Check-based (user-supplied check:)

Providers: bash, ansible

Probe method: The user supplies a `check:` shell command. If the command exits 0, the resource is `ok`;
if it exits non-zero, the resource needs to be applied.

```yaml
- urn: "urn:hams:bash:install-homebrew"
  step: Install Homebrew
  run: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  check: command -v brew
```

### Filesystem (path existence)

Providers: git-clone, file

Probe method: The provider checks whether the target path exists on disk. For git-clone, it also
verifies the remote URL matches.

```yaml
- urn: "urn:hams:git-clone:dotfiles"
  step: Clone dotfiles repository
  remote: https://github.com/zthxxx/dotfiles.git
  local_path: ~/Projects/dotfiles
  default_branch: main
```

## How Providers Are Ordered

During `hams apply`, providers execute in the order defined by `provider_priority` in your config.
The default order is designed so dependencies come first (e.g., `bash` runs before `Homebrew` so
you can bootstrap Homebrew via a bash step if needed):

```yaml
# ~/.config/hams/hams.config.yaml
provider_priority:
  - bash
  - Homebrew
  - apt
  - pnpm
  - npm
  - uv
  - go
  - cargo
  - vscode-ext
  - mas
  - git-config
  - git-clone
  - defaults
  - duti
  - ansible
```

<Callout type="info">
  Override the order in your store's `hams.config.yaml` or use `--only` / `--except` flags on `hams apply`
  to run only specific providers.
</Callout>
