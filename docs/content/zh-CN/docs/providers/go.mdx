# go install Provider

包装 `go install`，用来从源码装 Go 写的命令行工具。写 Go 的人都熟 —— `go install github.com/x/y@latest`
就能得到一个二进制扔到 `$GOBIN` 里，hams 把这个过程记下来方便重现。

**支持平台：** macOS、Linux

## 怎么用

```bash
# 装一个（用完整的 import path）
hams go install github.com/golangci/golangci-lint/v2/cmd/golangci-lint@latest

# 不指定版本就是 @latest，hams 会帮你补上
hams go install github.com/goreleaser/goreleaser

# 看看装了哪些
hams go list
```

`go install` 没有原生的 "uninstall" 概念 —— 你要卸的话 hams 会去删 `$GOBIN` 下对应的二进制文件。

## Hamsfile 长什么样

```yaml
# macOS/go.hams.yaml
schema_version: 1
provider: go

groups:
  - tag: go-tools
    items:
      - app: github.com/golangci/golangci-lint/v2/cmd/golangci-lint
        version: latest
        intro: Go linter aggregator.
      - app: golang.org/x/tools/cmd/goimports
        intro: Go import organizer.
      - app: github.com/goreleaser/goreleaser
        version: v1.25.1
        intro: Go release tool.
```

- `app` 是完整的 import path，跟 `go install` 里写的一样
- `version` 可选，不填默认 `latest`。想钉死版本就写具体 tag

## 状态探测

Go 没有官方的"列一下全局装了啥"的命令（`go install` 本身是无状态的），所以 hams 的做法是：

1. 把 `$GOBIN` 目录（默认 `$GOPATH/bin`，没设就是 `~/go/bin`）扫一遍
2. 对每个二进制跑 `go version -m <bin>` 读它的模块信息
3. 比对 Hamsfile 里声明的 import path

这样即使你换了机器、`$GOBIN` 里一堆二进制，hams 也能认出哪些是自己管的。

## 前提条件

机器上得有 `go` 命令。通常用 Homebrew / apt 装一下就行：

```yaml
# Homebrew.hams.yaml
- app: go
  intro: Go programming language.
```

然后在 `provider_priority` 里让 `Homebrew`（或 `apt`）跑在 `go` 前面。

## 关于版本和 `@latest`

`go install x@latest` 每次都会去请求一次模块代理拿最新版，hams 在 apply 的时候会把实际装的版本记到
`.state/` 里。所以下次 apply 只要 Hamsfile 里还是 `latest` 且 `.state/` 里有记录，
就不会重新拉网络 —— 除非你显式跑 `hams refresh --force` 或 `hams go install <app>@latest` 让它刷新。

<Callout type="tip">
  跟 Homebrew、apt 不一样，go install 装的工具没有自动升级通道。想升级就重新
  `hams go install x@latest`，hams 会对比记录的版本，如果真的是新版会重装。
</Callout>
